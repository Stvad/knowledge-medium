Property Filter Adapter Seam

The typed query engine already has richer semantics than the backlinks filter UI exposes. We need an extension seam that lets a property describe its filter affordance without hard-coding domain knowledge into BacklinkFilters.

Current Shape

  • codec.where means the stored JSON value can be directly compared.
  • where[name].target means a ref or refList can be traversed and filtered by target block properties.
  • The backlinks UI lists direct-comparable codecs and has a local daily-note-ref special case.
  • All registered schemas can support is set and is unset, because those compile to JSON presence checks.

Problem

The UI currently knows that a ref to a daily note should behave like a date. That is useful, but it leaks daily-notes semantics into a generic backlinks surface. It also missed Roam-imported date attributes that are represented as refList.

Proposed Seam

Add a property-filter adapter facet. Each adapter receives a registered schema and can return a compact description of how to render, parse, build, and format a filter for that property. The backlinks filter UI becomes a consumer of adapters, not a place where semantic property cases accumulate. 

interface PropertyFilterAdapter {
  id: string
  priority?: number
  match(schema: AnyPropertySchema): boolean
  operators(schema: AnyPropertySchema): PropertyFilterOperator[]
  input(schema: AnyPropertySchema, operator: string): PropertyFilterInput
  build(args: {
    schema: AnyPropertySchema
    propertyName: string
    operator: string
    rawValues: readonly string[]
  }): BlockPredicate | null
  format?(propertyName: string, value: unknown): string | null
}

Default Adapters

  • Presence adapter: matches every registered schema and provides is set / is unset.
  • Scalar adapter: matches schemas with codec.where and provides equality and scalar comparisons where appropriate.
  • Daily-note target adapter: matches ref and refList schemas targeting daily notes and builds target.daily-note:date predicates.

Ownership

Keep query capability in codecs and typed query. Put presentation and semantic shortcuts in adapters contributed by the owning feature. Daily notes owns daily-note-as-date behavior; Roam import can contribute compatibility adapters if imported vocabulary needs special UX.
